# AWS Support Case Reports

This repository contains simulated AWS Support Engineer case reports based on real-world cloud incidents.  
Each case follows a structured AWS Support workflow including investigation, root cause analysis (RCA), resolution, and preventive recommendations.

## Skills Demonstrated
- AWS EC2 troubleshooting
- CloudWatch monitoring and metrics analysis
- Incident response and root cause analysis (RCA)
- Networking and security group diagnostics
- Customer-focused technical communication
- Production issue documentation

## Case Reports
### Case 001 – EC2 Application Performance Degradation
- Diagnosed high CPU utilization using CloudWatch
- Identified under-provisioned instance as root cause
- Implemented remediation and preventive monitoring

### Case 002 – Database Connection Failure
- Investigated application connectivity issues
- Analyzed logs and network configurations
- Documented resolution and preventive actions

## Tools & Services Used
- Amazon EC2
- Amazon CloudWatch
- AWS VPC & Security Groups
- Linux (Amazon Linux 2)

## Role Alignment
This project is designed to reflect the responsibilities of an **AWS Support Engineer**, focusing on customer-impacting issues, clear communication, and structured problem resolution.
